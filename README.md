# mihomo_ConfigurationTemplate

# 说明

这几天有空花时间研究了一下mihomo的规则配置，写了几份mihomo配置文件模板出来，在每个重要的字段都写了中文注释便于理解与学习

使用官方的配置模板修改过来的，参考了很多大佬的配置，增加了一些优化项，所使用的域名和ip库也是使用官方模板自带的库

## 三份配置文件的区别
| 配置文件类型 | 说明 | 特点 |
|---|---|---|
| 本地GeoData.yaml | 本地数据库解析，适合网络不好的环境。每24个小时更新一次数据库，即使更新失败，也不影响正常使用。优点是稳定性好，缺点是GeoData数据库文件较大，格式老旧，解析速度一般，此方法官方已不推荐 | 速度：⭐⭐⭐<br>稳定：⭐⭐⭐⭐⭐ |
| 在线rule-set.yaml | 在线解析域名和ip库，每次启动都会把数据库写到缓存里，实时更新，msr数据库文件小，解析速度快。适合网络环境良好，追求速度的环境使用。优点是速度最快，缺点是依赖网络 | 速度：⭐⭐⭐⭐⭐<br>稳定：⭐⭐ |
| 在线mrsDomain+本地mmdbIP混合解析.yaml | 本地mrs域名库+在线mmdbip库混合解析，平衡速度和稳定的选择 | 速度：⭐⭐⭐⭐<br>稳定：⭐⭐⭐⭐ |



# 使用说明

以前一直都在使用第三方的客户端，启动速度慢不说，关键是还占位置占内存，经过学习，发现直接使用官方的内核就很方便使用了

1. 直接在[mihomo官网](https://github.com/MetaCubeX/mihomo)下载所需的可执行内核文件
2. 把配置文件命名为 `config.yaml` 和内核文件放在一个目录，
3. 然后命令执行 `mihomo.exe -d .` （win平台为例）就可以跑起来了
4. 其它的所有配置，都可以在`config.yaml`配置文件里修改，也可以访问[后台UI管理平台](http://localhost:9090/ui) `http://localhost:9090/ui` 进行修改和控制

至于Windows怎么隐藏命令行窗口启动，这个就不再赘述，方法多的是。可以用 vbs 脚本，也可以用 winsw 注册成服务
